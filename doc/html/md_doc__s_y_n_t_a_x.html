Syntax Reference

General

The PCC syntax looks like a mixture of Assembler, BASIC and C(++); that's why I like to call it an ABC-Syntax.

The syntax is strictly command-based (mnemonics) but often allows for a variable number of arguments.

  • Every line of a PCC script consists of a single command.
  • Commands are case insensitive, although I prefer to use all UPPERCASE.
  • Variable names and predefined identifiers, however, are case sensitive.
  • Arguments to commands are separated by a single whitespace character.
  • Additional whitespace (space and tab characters) is ignored.
  • Nested commands are not supported. (Always store intermediate results.)
  • Every argument has a data type which can dynamically change at runtime.

Use hashtag (#) comments to document your code. (Block comments are not supported.) E.g.:

# This is a one-line comment.
PRINT "Hi!" # This is an end-of-line comment.

Data types

Every argument has one of the following basic data types. Type abbreviations are introduced for convenience.

Basic types

Type Description
IDENT A variable name or label starts with a letter and may contain letters (a..z, A..Z), digits (0..9) and underscores (_)
IDENTARR A variable name followed by an additional pair of square brackets containing an arbitrary string, i.e. MyVariable[0123Hello]
STRING A constant character string is enclosed in double quotes ("Hi!"). To include a double quote inside the string itself, it must be escaped using a backslash (\"). To include a backslash inside the string, it must also be escaped using another backslash (\\).
NUMBER A constant number consists of digits and may contain a single dot for floating point values and a minus sign for negative values.

Type abbreviations

General types:

  • VALUE = NUMBER or STRING
  • IDVAL = IDENT or IDENTARR or VALUE
  • ANY = Arbitrary list of space-separated IDVAL type arguments

Literal types:

  • REFSTR = STRING or IDENT or IDENTARR, where the dereferenced variable (IDENT) must have type STRING.

Mathematical types:

  • REFNUM = NUMBER or IDENT or IDENTARR, where the dereferenced variable (IDENT or IDENTARR) must have type NUMBER.
  • OPERATOR = + or - or * or /

If a dereferenced variable (REFSTR or REFNUM) does not match the expected type, a runtime error will be thrown.

Command reference

The standard commands are categorized into several groups.

The notation of command-specific arguments introduces references (x) for easier documentation, like so: <ARGTYPE>(x)

General

Command Arguments Description
QUIT none Stop script execution / leave interactive session
HELP none Show the list of all commands
USER none Start nested interactive session. Useful for debugging. User variables will be shared between sessions.
START REFSTR Start script session using the supplied filename.

Variables

Command Arguments Description
SET IDENT(x) = ANY Load variable (x) with any IDVALs (1 or more). If just one IDVAL is supplied, the variable will assume its type. Multiple IDVALs will be concatenated as a STRING.
DUMP none Show the list of all user defined variables.

Console

Command Arguments Description
PRINT ANY Output any arguments to the console. Integer numbers are displayed as such while floating point numbers are displayed with six significant figures.
INPUT IDENT(x) Query user input from console & store in variable (x).

The input format must match the current data type of variable (x). On mismatch, the query will be repeated.

Operating System

Command Arguments Description
SYSTEM REFSTR Execute OS-specific command.

Mathematics

Mathematical operations are carried out in reverse polish notation (RPN) because it is easy to implement and it doesn't depend on braces or on operator precedence.

Operands must be either numbers or variables. Supplying a variable with a type other than NUMBER will result in a runtime error.

Command Operands Description
MATH IDENT(x) = RPN See table below

A simple RPN term consists of an operator followed by one or more operands. See the table below for a full list of supported operators and the required number of operands.

RPN terms may be nested arbitrarily, e.g. + * 4 5 3 is valid and is equivalent to ((4 * 5) + 3) in infix ("normal") notation.

The result of all (nested) calculations will then be stored in the variable (x).

Operator Operands Description
+ REFNUM(a) REFNUM(b) Add (a) to (b)
- REFNUM(a) REFNUM(b) Subtract from (a) the value (b)
* REFNUM(a) REFNUM(b) Multiply (a) times (b)
/ REFNUM(a) REFNUM(b) Divide (a) by (b)

You are hereby invited to also implement an infix notation parser, if that's what you like better. :-)

User variables

Variables consist of an identifier (IDENT) and a value (VALUE). All variables are global, i.e. they can always be accessed from any section of the program.

PCC variables have dynamic data types, i.e. variables will adopt the data type of the data assigned to it using the SET command.

Variables do not have to be declared explicitly. A variable (IDENT/VALUE pair) is created as soon as a value is assigned to it.

Please note that you have to assign a value to a variable first before you can read its value. Accessing an undefined variable will result in a runtime error.
